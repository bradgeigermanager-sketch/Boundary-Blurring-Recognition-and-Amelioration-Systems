Hybrid Architecture Diagram  
### Purpose  
A conceptual diagram showing how organizational and device boundaries should interlock without contamination.

Below is a **text‑based architecture diagram** suitable for system design documents.

# The Non‑Obvious Insight  
The hybrid architecture works because it **forces provenance to be explicit**.  
Boundary blurring is never about too much activity—it’s about activity with **unclear origin**.

Once origin is restored:

- Organizational intent becomes legible  
- Device behavior becomes predictable  
- Feedback loops become stable  
- Cross‑domain interactions become safe  

This is the foundation of any anomaly‑resistant system.
